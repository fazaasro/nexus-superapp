### 1. cAdvisor Fix - Container Healthy âœ…

**Problem:**
- cAdvisor container stuck in restart loop (Exit code: 2)
- Restarting repeatedly since previous day

**Root Cause Investigation:**
1. First attempt: "unsupported metric 'accelerator' specified"
   - `accelerator` is NOT a valid metric to disable in cAdvisor v0.47.2
   - Valid metrics: advtcp,app,cpu,cpuLoad,cpu_topology,cpuset,disk,diskIO,hugetlb,memory,memory_numa,network,oom_event,percpu,perf_event,process,referenced_memory,resctrl,sched,tcp,udp

2. Second attempt: "unknown backend storage driver: docker"
   - `--storage_driver=docker` is NOT a valid option in cAdvisor v0.47.2
   - Valid options: <empty>, bigquery, elasticsearch, influxdb, kafka, redis, statsd, stdout

**Fix Applied:**
- Removed `--storage_driver=docker` flag (use default empty value)
- Removed `accelerator` from `--disable_metrics` list
- Used `docker compose up -d --force-recreate cadvisor` to apply changes
  - Note: `restart` command doesn't apply config changes!

**Result:**
- cAdvisor now healthy and serving metrics
- All monitoring stack containers healthy
- Git commit: "fix(cadvisor): remove invalid storage_driver and accelerator metric"

**Files Modified:**
- `/home/ai-dev/swarm/repos/overseer/docker-compose.monitoring.yml`
  - Line 239: Removed `--storage_driver=docker`
  - Line 245: Removed `accelerator` from disabled metrics

**Lessons Learned:**
- Always check documentation for exact flag values
- cAdvisor v0.47.2 has specific valid options for storage_driver
- Use `docker compose up -d --force-recreate` instead of `restart` to apply config changes
- Git commit message should be descriptive and follow conventional commits
